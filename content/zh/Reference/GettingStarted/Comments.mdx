# 评论

在本课程的代码示例中，您可能会注意到以 `#` 符号开头的行。这些称为**注释**。


```python
# This is a comment in Python
print("Hello, World!")
```

注释用于在代码中添加解释或注解。Python 解释器会忽略它们，这意味着它们不会影响程序的运行。

在这些代码示例中包含注释是为了说明问题。然而，一般来说，您不应该养成在代码中写注释的习惯。这有几个原因。

## 为什么要避免在代码中编写注释

**1.如果你的代码写得很好，不言自明，你就不需要通过注释来理解它的作用。事实上，如果使用不当，注释有时会让代码更难读**。

这是一个不言自明、需要注释才能理解的代码示例：

```python
# p is the price of the item
p = 20

# r is the sales tax rate (7%)
r = 0.07

# t is the amount of tax
t = p * r

# f is the final total price
f = p + t

# print the final total
print(f)
```

这是相同的代码，但编写方式更简单明了，不需要注释：

```python
price = 20
sales_tax_rate = 0.07
tax_amount = price * sales_tax_rate
final_total = price + tax_amount
print(final_total)
```

你看，第二个版本其实更简洁易懂，不需要注释来解释每个变量代表什么？

通常，创建清晰且不言自明的代码只需选择好的变量名和函数名，并以合理的方式组织代码即可。这种方法比依靠注释来解释代码要好得多。

**2.如果您修改了代码却忘记更新注释，注释就会过时或不正确。这可能会给日后阅读您代码的人带来困惑**。

试想一下，现在的代码应该在应用折扣后计算最终总价，但您却忘记更新注释：


```python
# p is the price of the item
p = 20

# r is the sales tax rate (7%)
r = 0.20

# t is the amount of tax
t = p * r

# f is the final total price
f = p - t

# print the final total
print(f)
```

现在的注释具有误导性且不正确，可能会给以后阅读代码的人造成困惑。他们可能会认为代码
- 本应在应用销售税后计算最终总价，但代码中存在一个错误，实际上是应用了折扣
- 本应在应用折扣后计算最终总价，但代码中存在一个错误，实际应用的是销售税

##当注释有用时

不过，在一些边缘情况下，注释也会有所帮助：

**1.解释为什么要以某种方式做某事或做出某个特定决定，尤其是在代码本身并不明显的情况下**。


```python
age = 16

# Add 1 because the form asks for the age the student will turn this year
age = age + 1

print(age)
```

在这里，我们使用注释来解释一个看似有点奇怪的选择（在年龄上加 1），这对于阅读代码的人来说可能不是很明显。这有助于防止混淆，并清楚地说明我们为什么要以某种方式做某些事情。

**2.在测试或调试时暂时禁用一行代码（称为 "注释 "代码）。不过，这只能用于短期测试，不应留在最终代码中**。


```python
# This line of code is commented out and will not run
# print("This line is commented out and will not be executed")
print("This line will be executed")
```

如果您确实需要使用注释，请确保您有计划不断更新它们，并确保它们能为您的代码增加价值，而不仅仅是使代码变得杂乱无章。
