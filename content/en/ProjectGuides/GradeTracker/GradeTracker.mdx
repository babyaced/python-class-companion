# Project 2: Grade Tracker

You'll build a program that stores grades in a list, computes stats (average, highest, lowest), and later saves and loads grades from a file. The project is split into three phases that match the topics you learn each day.

---

## Phase 1 (Day 1): Lists and Loops

**Learning focus:** Create a list, loop through the list to display and to compute a sum.

**Tasks:**

<Checkbox
  id="proj2-phase1-task1"
  label="Create a static list of grades in your code, for example grades = [85, 92, 78, 88]. This will be the list you use for the rest of the tasks."
/>

<Checkbox
  id="proj2-phase1-task2"
  label="Write a for loop that prints each grade in the list (one per line or all on one line)."
/>

<Checkbox
  id="proj2-phase1-task3"
  label="Use a for loop to add up all the grades into a variable and print the sum."
/>

<Checkbox
  id="proj2-phase1-task4"
  label="Compute and print the average grade using the sum and len(grades)."
/>

**Topics covered:** [Lists](../../Variables/TypesOfVariables/DataStructures/Lists), [Loops](../../ControlFlow/Loops)

**End of Day 1:** You have a program with a list of grades that prints them and shows the sum and average.

**Challenge (optional):** Instead of a static list, add grades using a loop. For example: use a **for** loop that runs 5 times and asks for a grade with `input()` each time, appending the result to the list (convert to float).

---

## Phase 2 (Day 2): Conditionals

**Learning focus:** Use `if` / `elif` / `else` to make decisions based on the grades.

**Tasks:**

<Checkbox
  id="proj2-phase2-task1"
  label="Loop through the list of grades, track the highest grade seen so far in a variable like highest, and print the highest grade at the end."
/>

<Checkbox
  id="proj2-phase2-task2"
  label="Loop through the list again (or in the same loop) to track and print the lowest grade."
/>

<Checkbox
  id="proj2-phase2-task3"
  label='Based on the average grade, use if/elif/else to print a message (for example: 90+ → "Great job!", 70–89 → "You&apos;re on track.", below 70 → "Let&apos;s work on bringing this up.").'
/>

**Topics covered:** [If statements](../../ControlFlow/IfStatements), [Comparison operators](../../ControlFlow/ComparisonOperators), [Loops](../../ControlFlow/Loops)

**Challenge** (optional):   Loop through the list and print only the grades that are below 70 (or another number you choose). This practices "filtering" with an `if` inside a loop. Add a print statement to label these grades as "Needs Improvement".

**End of Day 2:** Your program shows highest, lowest, a message based on the average, and optionally which grades need work.

---

## Phase 3 (Day 3): File I/O

**Learning focus:** Save the grades list to a file and load it back so data persists between runs.

**Before you start:** Create a file named `grades.txt` in the same folder as your program. You can leave it empty, or put a few grades in it (one number per line). That way the file exists when your program runs—no error handling needed for this project.

**Tasks:**

<Checkbox
  id="proj2-phase3-task1"
  label="Write code (or a function) that opens your file grades.txt for writing and writes each grade on its own line, then closes the file."
/>

<Checkbox
  id="proj2-phase3-task2"
  label="Write code (or a function) that opens grades.txt for reading, converts each line to a float, appends it to a list, and returns or uses that list as your grades list."
/>

<Checkbox
  id="proj2-phase3-task3"
  label="When the program starts, load grades from the file; at the end, after showing stats, save the current list back to the file so data persists between runs."
/>

**Topics covered:** [Handling files](../../InputAndOutput/HandlingFiles), [Lists](../../Variables/TypesOfVariables/DataStructures/Lists), [Loops](../../ControlFlow/Loops)

**End of Day 3:** You have a full grade tracker: load grades from the file, see stats (sum, average, highest, lowest, messages), and save the list back to the file so it persists next time.

---

## Summary

| Day  | Focus           | What you build |
|------|-----------------|----------------|
| 1    | Lists + Loops   | List of grades, add/display, sum, average |
| 2    | Conditionals    | Highest, lowest, message based on average, optional filter |
| 3    | File I/O        | Save and load grades so they persist |

You can add small refinements (e.g. a simple menu: "1) Add grades 2) Show stats 3) Save 4) Quit") if there's time, but the core is lists, loops, conditionals, and file I/O in three digestible phases.
