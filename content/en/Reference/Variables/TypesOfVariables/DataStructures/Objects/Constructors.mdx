# Constructors

When you create an object from a class, Python automatically runs a special method called the **constructor**. Its job is to set up the object's starting values.

## `__init__`

In Python, the constructor is always named `__init__`. It runs the moment you create a new object:

```python
class Dog:
    def __init__(self, name, age):
        self.name = name
        self.age = age
```

When you write `Dog("Buddy", 3)`, Python calls `__init__` automatically — you never call it yourself.

## Understanding `self`

`self` refers to the specific object being created. When Python runs `__init__`, it passes the new object in as `self` so the method knows *which* dog it's setting up.

```python
dog1 = Dog("Buddy", 3)
dog2 = Dog("Max", 5)
```

When `dog1` is created, `self` is `dog1`. When `dog2` is created, `self` is `dog2`. That's how each object ends up with its own `name` and `age` — `self.name` and `self.age` are stored on that specific instance.

You don't pass `self` yourself — Python handles it automatically.
