# Project 3: Generative Art

In this project you'll build a program that generates original artwork using code. By the end, your program will place random shapes on a canvas and save the result as an image file.

You'll build it one piece at a time over six classes — each class adds something new to the project.

---

# Class 1 — Define Your Color Palette

**What you'll use:** [Lists](/en/Reference/Variables/TypesOfVariables/DataStructures/Lists/page), [Dictionaries](/en/Reference/Variables/TypesOfVariables/DataStructures/Dictionaries/page)

Your first task is to define the colors your artwork will use. Colors in Pillow are tuples of `(red, green, blue)` — each value is between 0 and 255.

You can use [Coolors](https://coolors.co/) to generate a palette, but you'll need to convert the hex codes to RGB tuples (e.g. `#FF5733` → `(255, 87, 51)`). [This hex to RGB converter](https://www.rapidtables.com/convert/color/hex-to-rgb.html) can help.

# Class 3 — Shape Classes

**What you'll use:** [Classes](/en/Reference/Variables/TypesOfVariables/DataStructures/Objects/page)

Right now your drawing code is loose — position, size, and color are just separate variables floating around. Let's fix that by creating classes that bundle a shape's data and its drawing logic together.

Define a `Circle` class and a `Rectangle` class. Each one should know its own position, size, and color — and how to draw itself onto a canvas.

**Tasks:**

<Checklist>
  <Checkbox id="ga-s3-1" label="Define a Circle class with __init__ (x, y, radius, color) and a draw(draw_context) method." />
  <Checkbox id="ga-s3-2" label="Define a Rectangle class with __init__ (x, y, width, height, color) and a draw(draw_context) method." />
  <Checkbox id="ga-s3-3" label="Create one Circle and one Rectangle manually and draw them to test both work." />
</Checklist>

<Details>
  <summary>Hint — structuring a shape class</summary>

  ```
  class Circle:
      init takes: x, y, radius, color
          store each as an attribute (self.x, self.y, etc.)

      draw takes: draw_context
          use draw_context to draw an ellipse using self.x, self.y, self.radius, self.color
  ```
</Details>

# Class 4 — The Canvas Class

**What you'll use:** [Classes](/en/Reference/Variables/TypesOfVariables/DataStructures/Objects/page), [Lists](/en/Reference/Variables/TypesOfVariables/DataStructures/Lists/page), [Loops](/en/Reference/ControlFlow/Loops/page)

Now build a `Canvas` class that stores a collection of shapes and renders them all. Here's the key idea: Canvas doesn't need to know whether a shape is a `Circle` or a `Rectangle` — it just calls `draw()` on each one and they handle the rest.

**Tasks:**

<Checklist>
  <Checkbox id="ga-s4-1" label="Define a Canvas class with __init__ (width, height, background_color), add_shape, and render methods." />
  <Checkbox id="ga-s4-2" label="Have Canvas store shapes in a list. render() should loop through the list and call draw() on each shape." />
  <Checkbox id="ga-s4-3" label="Create a Canvas, add a mix of Circle and Rectangle objects (use a loop or random generation), and call render() to produce an image." />
  <Checkbox id="ga-s4-4" label="Use random to generate at least 50 shapes with randomized attributes." />
</Checklist>

<Details>
  <summary>Hint — the render loop</summary>

  ```
  render:
      create a draw context from the image
      for each shape in self.shapes:
          call shape.draw(draw_context)
      save the image
  ```

  Notice that render doesn't check what type of shape it has — it just calls draw(). This works because both Circle and Rectangle have a draw() method.
</Details>
